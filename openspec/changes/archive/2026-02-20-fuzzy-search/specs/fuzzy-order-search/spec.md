## ADDED Requirements

### Requirement: 用户可以按订单号模糊搜索订单
系统应当支持通过订单号的部分匹配来搜索订单。

#### Scenario: 订单号部分匹配成功
- **WHEN** 用户传入订单号关键词 "2024"
- **THEN** 系统返回所有订单号中包含 "2024" 的订单列表

#### Scenario: 订单号为空时返回所有订单
- **WHEN** 用户不传入订单号参数或传入空字符串
- **THEN** 订单号条件不参与过滤，返回符合其他条件的订单

### Requirement: 用户可以按客户姓名模糊搜索订单
系统应当支持通过关联客户姓名的部分匹配来搜索订单。

#### Scenario: 客户姓名匹配订单
- **WHEN** 用户传入客户姓名 "张"
- **THEN** 系统返回所有客户姓名中包含 "张" 的订单列表

### Requirement: 用户可以按衣物类型模糊搜索订单
系统应当支持通过订单中衣物类型的关键字搜索订单。

#### Scenario: 衣物类型关键字匹配
- **WHEN** 用户传入关键词 "衬衫"
- **THEN** 系统返回所有包含 "衬衫" 衣物的订单列表

### Requirement: 用户可以组合多条件模糊搜索订单
系统应当支持同时使用多个条件进行模糊搜索，各条件之间为 AND 逻辑。

#### Scenario: 订单号 + 客户姓名组合搜索
- **WHEN** 用户同时传入订单号 "2024" 和客户姓名 "张"
- **THEN** 系统返回订单号包含 "2024" 且客户姓名包含 "张" 的订单列表

#### Scenario: 多条件组合无匹配
- **WHEN** 用户传入的条件组合没有任何订单同时满足
- **THEN** 系统返回空列表

### Requirement: 订单模糊搜索接口返回标准响应格式
系统应当返回与现有接口一致的响应格式。

#### Scenario: 成功响应
- **WHEN** 模糊搜索执行成功
- **THEN** 返回 HTTP 200 和订单列表（List<Order>），空列表时返回 []

#### Scenario: 返回的订单包含关联客户信息
- **WHEN** 订单搜索返回结果
- **THEN** 每条订单记录应包含关联的客户 ID 信息
