## ADDED Requirements

### Requirement: 用户可以按姓名模糊搜索客户
系统应当支持通过客户姓名的部分匹配来搜索客户，使用 LIKE 查询。

#### Scenario: 姓名部分匹配成功
- **WHEN** 用户传入姓名关键词 "张"
- **THEN** 系统返回所有姓名中包含 "张" 的客户列表（如 "张三"、"小张"、"张老三"）

#### Scenario: 姓名为空时返回所有客户
- **WHEN** 用户不传入姓名参数或传入空字符串
- **THEN** 姓名条件不参与过滤，返回符合其他条件的客户

#### Scenario: 特殊字符处理
- **WHEN** 用户传入包含空格的姓名 "王 五"
- **THEN** 系统正常执行 LIKE 查询，返回匹配结果

### Requirement: 用户可以按手机号模糊搜索客户
系统应当支持通过手机号的部分匹配来搜索客户。

#### Scenario: 手机号后几位匹配
- **WHEN** 用户传入手机号后四位 "1234"
- **THEN** 系统返回所有手机号中包含 "1234" 的客户列表

#### Scenario: 手机号前缀匹配
- **WHEN** 用户传入手机号前三位 "139"
- **THEN** 系统返回所有以 "139" 开头的客户列表

### Requirement: 用户可以按备注模糊搜索客户
系统应当支持通过客户备注信息的关键字搜索。

#### Scenario: 备注关键字匹配
- **WHEN** 用户传入关键词 "VIP"
- **THEN** 系统返回所有备注中包含 "VIP" 的客户列表

### Requirement: 用户可以组合多条件模糊搜索客户
系统应当支持同时使用多个条件进行模糊搜索，各条件之间为 AND 逻辑。

#### Scenario: 姓名 + 手机号组合搜索
- **WHEN** 用户同时传入姓名 "张" 和手机号 "139"
- **THEN** 系统返回姓名包含 "张" 且手机号包含 "139" 的客户列表

#### Scenario: 多条件之一无匹配
- **WHEN** 用户传入的条件组合没有任何客户同时满足
- **THEN** 系统返回空列表

### Requirement: 模糊搜索接口返回标准响应格式
系统应当返回与现有接口一致的响应格式。

#### Scenario: 成功响应
- **WHEN** 模糊搜索执行成功
- **THEN** 返回 HTTP 200 和客户列表（List<Customer>），空列表时返回 []
