## ADDED Requirements

### Requirement: 用户可以查看日报
系统应当支持查看指定日期的收入统计和订单明细。

#### Scenario: 查看今日统计
- **WHEN** 用户打开统计页面
- **THEN** 系统默认显示今日的収入、现金收入、储值收入、未完成订单数

#### Scenario: 查看指定日期统计
- **WHEN** 用户选择日期并查询
- **THEN** 系统显示该日期的收入统计和订单明细

#### Scenario: 订单明细显示
- **WHEN** 查看订单明细
- **THEN** 显示订单号、客户姓名、金额、支付方式、状态、时间

### Requirement: 用户可以查看月报
系统应当支持查看指定月份的收入统计和本月订单明细。

#### Scenario: 查看本月统计
- **WHEN** 用户打开月报页面
- **THEN** 系统默认显示本月的总收入

#### Scenario: 查看指定月份统计
- **WHEN** 用户选择年份和月份并查询
- **THEN** 系统显示该月份的收入统计和本月所有订单明细

### Requirement: 收入统计计算正确
系统应当正确计算各类收入。

#### Scenario: 今日收入计算
- **WHEN** 查看今日收入
- **THEN** 显示现金收入 + 储值充值金额（不包括储值支付订单）

#### Scenario: 现金收入计算
- **WHEN** 查看现金收入
- **THEN** 显示当天现金支付的订单总金额

#### Scenario: 储值收入计算
- **WHEN** 查看储值收入
- **THEN** 显示当天客户充值的总金额
